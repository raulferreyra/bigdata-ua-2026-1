## ¿Qué aprendí?
Aprendí que Big Data sirve para procesar millones de datos desordenados (como texto de redes sociales o logs) que una base de datos normal como SQL Server no puede aguantar en tiempo real. También aprendí que herramientas como Kafka sirven para recibir el golpe de datos rápido y Spark para procesarlos en memoria de inmediato. Esto permite automatizar cosas como detectar fraudes al instante y recomendar productos mientras el cliente compra.

## ¿Qué fue lo más difícil?
Calcular el volumen de datos de tantos millones de transacciones al día y entender cómo meter datos que no tienen filas ni columnas (como las quejas de internet o audios) dentro de una misma arquitectura sin que el sistema se vuelva lento.

## ¿Cómo aplica en mi empresa actual?
Aplica para sacar los reportes pesados, las tramas de seguros y los archivos de texto fuera del servidor principal de la empresa. Al moverlos a un almacén separado, podemos hacer análisis de datos y estadísticas sin colapsar ni volver lento el sistema que los usuarios usan para trabajar en el día a día.
